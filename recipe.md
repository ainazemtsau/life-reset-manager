# Рецепт — текущая конфигурация работы (редактируемая)

Это «как мы сейчас работаем». Менеджер меняет ЭТОТ файл через ворота (с «ок» владельца), по
итогам review / deep-research. `manager.md` (как вести процесс) и `sealed-core.md` (пол
безопасности) — стабильны; меняется в основном рецепт.

## Метод
- current: **НЕ ЗАДАН** — задаётся процедурой 4.1 (подбор метода через deep-research на первом
  запуске; не берём по умолчанию).
- history: (сюда — когда и почему метод менялся)

## Компоненты дня (по порядку)
Каждый компонент: имя — что делает (вход → выход). Это стартовый скелет; deep-research может его
пересобрать.
- **ворота** — вчерашнее review? во сколько встал; floors. (вчерашний review → день открыт)
- **работа-по-методу** — выполнить главное дня по текущему методу. (главное из недели → лог по ходу)
- **дневник** — *пример компонента*: вопросы дня. (ответы владельца → запись) [включается решением review]
- **review-дня** — что сделано/нет, floors, 1 вывод. (→ сохранённый review)
